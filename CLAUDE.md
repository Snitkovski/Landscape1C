## О проекте

Интерактивный ландшафт (карта) технологий экосистемы 1С — наследник линейного каталога [StackTechnologies1C](https://github.com/Oxotka/StackTechnologies1C). Ключевая идея — навигация по **двум независимым осям**: функциональная роль и контекст работы. Проект на стадии прототипа. UI **только русский** — все метки, значения фильтров и контент пишутся по-русски.

## Архитектура

**Живой сайт — это `app/`: статика без сборки** (HTML/CSS/JS, без бандлера, без зависимостей).

- `app/data.js` — единственный источник данных сайта: глобальный объект `window.LANDSCAPE` (не `fetch`, чтобы работало и через `file://`, и на GitHub Pages). Содержит `categories`, `axes` (определения фильтров) и `items` (карточки).
- `app/app.js` — вся логика: рендер фильтров/наборов/доски, фильтрация, детальная модалка, синхронизация состояния с URL (`?role=…&context=…&q=…`) для шаринга.
- `app/styles.css`, `app/index.html`, `app/logos/` — оформление и логотипы (SVG, имя файла указывается в `item.logo`; `null` → заглушка «1С»).

**Модель фильтрации (`app/app.js`).** Пять осей: `role`, `context`, `maturity`, `origin`, `license`. `role` и `context` в данных — массивы (`roles`, `contexts`), остальные — скаляры (`maturity`, `origin`, `license`). Внутри одной оси выбранные значения объединяются по ИЛИ, между осями — по И. «Стартовый набор» = выбрать один контекст и сбросить остальные оси.

**Инвариант значений.** Строковые значения в `item` (например `maturity: "продвинутое"`, `roles: ["devops"]`) должны точно совпадать со значениями в `axes.<ось>.values` в `app/data.js` — иначе фильтр/чип не сработает. При добавлении нового значения метки правь и `axes`, и элементы.

## Два формата данных (важно)

Данные существуют в двух местах, и они **не синхронизируются автоматически**:

- `app/data.js` — **рабочие данные сайта** (формат JS, поля `roles/contexts/maturity/origin/license/why/start/...`). Правки контента вносить сюда.
- `data/landscape.yml` + `data/settings.yml` — исходный формат движка **landscape2** (доменные поля в блоке `extra`). Это **документированный результат разведки**, в деплое не используется. Конвертация в `app/data.js` сейчас выполнена вручную (скрипта-парсера нет).

`build/` — артефакт сборки landscape2 (в `.gitignore`), тоже не часть живого сайта.

## Команды

Локальный предпросмотр (сборки нет):

```bash
cd app && python3 -m http.server 8123
# http://127.0.0.1:8123/
```

Проверка целостности данных (оси, блоки, файлы логотипов, связи, буква «ё»):

```bash
node scripts/validate.js
```

Деплой — GitHub Actions (`.github/workflows/deploy.yml`), выкладывает содержимое `app/` на GitHub Pages. Сейчас **только ручной запуск** (`workflow_dispatch`); автодеплой по push в `main` закомментирован (репозиторий приватный). Пути в `app/` относительные — работает на projectpage без доп. настроек.

## Соглашения

- Юнит-тестов нет; целостность данных проверяет `node scripts/validate.js`, вёрстка — визуально через локальный предпросмотр.
- Тёмные (монохромные) логотипы помечаются `"logoInvert": true` в `app/data.js` — движок сам инвертирует их в тёмной теме (класс `is-invert`).
- Стиль кода в `app/app.js` лаконичный: короткие хелперы (`$`, `cap`), секции под `// ──` комментариями. Держись этого стиля.
