> **Сначала прочитай [`docs/TZ.md`](docs/TZ.md).** Это главный документ проекта — техническое задание, замысел, принятые архитектурные решения и планы (две оси, шкала зрелости, связи между технологиями и граф, экспертный совет, слой компетенций, опрос популярности). Перед любыми содержательными изменениями сверяйся с ним и держи его в актуальном состоянии.

## О проекте

Интерактивный ландшафт (карта) технологий экосистемы 1С — наследник линейного каталога [StackTechnologies1C](https://github.com/Oxotka/StackTechnologies1C). Ключевая идея — навигация по **двум независимым осям**: функциональная роль и контекст работы. UI **только русский** — все метки, значения фильтров и контент пишутся по-русски.

## Архитектура

**Живой сайт — это `app/`: статика без сборки** (HTML/CSS/JS, без бандлера, без зависимостей).

- `app/data.js` — единственный источник данных сайта: глобальный объект `window.LANDSCAPE` (не `fetch`, чтобы работало и через `file://`, и на GitHub Pages). Содержит `categories`, `blocks` (группировка категорий), `axes` (определения фильтров) и `items` (карточки).
- `app/app.js` — главная страница: рендер фильтров и доски, фильтрация, сортировка, синхронизация состояния с URL (`?role=…&context=…&q=…`) для шаринга.
- `app/detail.js` — общий модуль детальной модалки (`window.openDetail(item)`), переиспользуется и главной, и графом. Здесь же блоки «Аналоги»/«Зависимости» и кнопки связей.
- `app/graph.html` + `app/graph.js` — отдельная страница «Граф связей»: force-directed canvas-граф по `analogs`/`depends`, клик по узлу открывает ту же модалку.
- `app/scheme.html` + `app/scheme.js` — страница «Схема»: статичный постер-древо (блок → категория → карточки инструментов) целиком в одном инлайн-SVG. Отбор по роли/зрелости и скрытие блоков (чекбоксы «Блоки»), клик по карточке открывает ту же `openDetail`-модалку. Экспорт SVG/PNG/PDF/майндмап (.mm) — без зависимостей: PNG/PDF через canvas (PDF = JPEG в минимальном PDF), .mm = FreeMind XML из того же `currentTree()`. Логотипы вшиваются в SVG (векторные — инлайн с `clipPath` по `viewBox`, чтобы открывалось в Illustrator; растровые — data-URI). На мобильном отборы прячутся в попап «Отборы» (триггер в шапке), «Скачать» → иконка «Поделиться».
- `app/council.html` — экспертный совет (кураторы + автор, фото в `app/people/`); `app/methodology.html` — методология разметки.
- `app/nav.js` — общая «обвязка» навигации для всех страниц: из единого конфига (список ссылок в начале файла) генерирует бургер-кнопку, выпадающее меню (навигация + Поддержать/Telegram + выбор темы светлая/темная/системная) и ссылки подвала. Править ссылки нужно только здесь. Отдельной иконки темы в шапке больше нет — тема выбирается только в бургер-меню (`.theme-toggle` скрыт глобально в CSS).
- `app/styles.css`, `app/index.html`, `app/logos/` — оформление и логотипы (SVG, имя файла указывается в `item.logo`; `null` → заглушка «1С»).

**Модель фильтрации (`app/app.js`).** Шесть осей: `role`, `context`, `maturity`, `origin`, `license`, `availability`. `role` и `context` в данных — массивы (`roles`, `contexts`), остальные — скаляры (`maturity`, `origin`, `license`, `availability`). Внутри одной оси выбранные значения объединяются по ИЛИ, между осями — по И. Курируемых «стартовых наборов» (kit-кнопок) больше нет — их роль выполняет фильтр по контексту.

**Инвариант значений.** Строковые значения в `item` (например `maturity: "продвинутое"`, `roles: ["devops"]`) должны точно совпадать со значениями в `axes.<ось>.values` в `app/data.js` — иначе фильтр/чип не сработает. При добавлении нового значения метки правь и `axes`, и элементы.

## Источник данных

Единственный источник — `app/data.js` (`window.LANDSCAPE`, поля `roles/contexts/maturity/origin/license/availability/why/start/analogs/depends/aliases/...`). Правки контента вносить сюда — вручную или через редактор (`app/editor.html`, см. ниже).

**`aliases`** — скрытый от пользователя массив поисковых синонимов (русские транслитерации англоязычных названий и пр.: `git`→`гит`, `SonarQube`→`сонаркуб`). Не отображается на карточке, участвует только в поиске (`app.js` → `matches()` подмешивает его в строку поиска). Редактируется в форме редактора (поле «Синонимы для поиска», через запятую).

## Команды

Локальный предпросмотр (сборки нет):

```bash
cd app && python3 -m http.server 8123
# http://127.0.0.1:8123/
```

Редактор разметки (`app/editor.html`) с прямой записью в `app/data.js` по кнопке «Сохранить» — нужен пишущий сервер (`python3 -m http.server` записывать не умеет):

```bash
python3 scripts/serve.py
# http://127.0.0.1:8123/editor.html  (отдаёт app/, принимает POST /save → пишет app/data.js)
```

`editor.js` сериализует весь `window.LANDSCAPE` (включая `updated`, `blocks`, все 6 осей) обратно тем же 2-пробельным JSON; при сохранении дата `updated` ставится автоматически. Без сервера «Сохранить» откатывается на скачивание копии.

Проверка целостности данных (оси, блоки, файлы логотипов, связи, буква «ё»):

```bash
node scripts/validate.js
```

Сборка `dist/` для ручной выгрузки на хостинг (сначала прогоняет валидатор, затем копирует публичную статику из `app/`, выкидывая редактор `editor.*` и `.DS_Store`):

```bash
node scripts/build.js
```

Деплой — GitHub Actions (`.github/workflows/deploy.yml`), выкладывает содержимое `app/` на GitHub Pages. Сейчас **только ручной запуск** (`workflow_dispatch`); автодеплой по push в `main` закомментирован (репозиторий приватный). Пути в `app/` относительные — работает на projectpage без доп. настроек.

## Соглашения

- Юнит-тестов нет; целостность данных проверяет `node scripts/validate.js`, вёрстка — визуально через локальный предпросмотр.
- Краткое описание (`description`) и «зачем нужно» (`why`) не заканчиваются точкой — последнее предложение пишется без точки. Это правило проверяет `validate.js`.
- Тёмные (монохромные) логотипы помечаются `"logoInvert": true` в `app/data.js` — движок сам инвертирует их в тёмной теме (класс `is-invert`).
- Стиль кода в `app/app.js` лаконичный: короткие хелперы (`$`, `cap`), секции под `// ──` комментариями. Держись этого стиля.
- Форматирование — Prettier (`.prettierrc`, `tabWidth: 4`); `app/*.{html,css,js}` прогнаны через него. После правок прогоняй `npx prettier --write app/<файл>`, чтобы не плодить шумные диффы. Исключение — `app/data.js` (в `.prettierignore`): его пишет редактор 2-пробельным JSON, Prettier его не трогает.
- Контентные страницы (схема, методология, экспертный совет) помечены `class="page-doc"` на `<body>`: на мобильном (`@media max-width 720px`) большой `masthead` скрыт, прилепленная шапка (`.topbar`) показана сразу, подвал не показывается. Версии в `?v=` у `styles.css`/`scheme.js` поднимаются вручную при правках (cache-bust на GitHub Pages).
