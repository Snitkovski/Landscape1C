# Бот-опросник популярности (прототип)

Телеграм-бот для сбора данных о реальной востребованности инструментов ландшафта
(замысел — `docs/TZ.md`, п.13). Живет отдельно от сайта: собирает ответы волнами
по годам, публикация на карточках — отдельный шаг после проверки данных.

## Запуск

1. Создать бота у [@BotFather](https://t.me/BotFather) (`/newbot`), получить токен.
2. Запустить (зависимостей нет, нужен только Node):

```bash
BOT_TOKEN=<токен> node bot/bot.js
```

3. Сводка по собранному (метрики State of JS: узнаваемость, использование, retention, интерес):

```bash
node bot/report.js               # все респонденты
node bot/report.js разработчик   # срез по роли респондента
```

## Как устроен опрос

- Онбординг: роль → уровень → контекст (значения — из осей `app/data.js`).
- Ядро сессии — инструменты своей роли: сначала базовые, потом продвинутые и
  нишевые, внутри уровня зрелости порядок случайный.
- Ответ про инструмент: «работал / слышал / не знаю»; после «работал» —
  опциональный сентимент «использовал бы снова?», после «слышал» — «хочешь попробовать?».
- После ядра — опт-ин блоки других ролей (не потоком: кому неинтересно, не видит).
- Прерваться можно всегда: состояние в `state.json`, при `/start` отвеченное не
  показывается повторно.

## Данные

- `answers.jsonl` (в `.gitignore`) — по строке на ответ:
  `{ts, wave, uid, role, level, context, block, tool, answer, sentiment}`.
  `role` — роль респондента, `block` — чей блок инструментов проходился,
  `uid` — хеш телеграм-аккаунта (соль — `BOT_SALT`, для боевого запуска задать свою).
- `state.json` (в `.gitignore`) — состояние сессий для продолжения.
- Хранилище изолировано в `saveAnswer`/`saveState` в `bot.js` — замена на БД
  (например, MongoDB) не трогает логику опроса.
